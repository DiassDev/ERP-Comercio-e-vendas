# ERP Acadêmico — Setor de Comércio e Vendas

## 📌 Sobre o Projeto

Este projeto faz parte de uma iniciativa acadêmica da **UGB-FERP (Centro Universitário Geraldo Di Biase)** que tem como objetivo simular um ambiente real de desenvolvimento de software baseado em **metodologias modernas de trabalho em equipe**, especialmente o modelo de **Squads** utilizado em empresas de tecnologia.

A proposta consiste na construção de um **sistema ERP modular**, onde cada grupo de alunos é responsável por desenvolver um **setor específico do sistema**, simulando a divisão de responsabilidades encontrada em projetos corporativos.

Nosso grupo é responsável pelo **módulo de Comércio/Vendas**, que representa o setor responsável pela gestão de transações comerciais dentro do ERP.

---

## 🎯 Objetivo

O objetivo deste módulo é desenvolver funcionalidades relacionadas à **gestão de vendas e operações comerciais**, permitindo o controle e registro de atividades fundamentais para empresas que realizam vendas de produtos ou serviços.

Além do desenvolvimento técnico, o projeto busca proporcionar experiência prática em:

* Trabalho em equipe no formato **Squad**
* Organização de projetos de software
* Versionamento de código com **Git**
* Estruturação de sistemas modulares
* Simulação de um ambiente profissional de desenvolvimento

---

## 🏢 Estrutura do Projeto Acadêmico

O ERP é dividido em diferentes **setores (módulos)**, onde cada grupo de alunos atua como um **time independente de desenvolvimento**, responsável por projetar, desenvolver e manter seu respectivo módulo.

Exemplos de setores dentro do ERP:

* Financeiro
* Recursos Humanos
* Logística
* Compras
* Comércio / Vendas (nosso módulo)

Essa divisão simula a arquitetura de sistemas empresariais reais, onde diferentes equipes trabalham em **componentes integrados do sistema**.

---

## 🛒 Módulo de Comércio / Vendas

O módulo de **Comércio/Vendas** é responsável por gerenciar operações comerciais dentro do ERP.

### Funcionalidades previstas

* Cadastro de clientes
* Registro de vendas
* Controle de pedidos
* Histórico de transações
* Consulta de vendas realizadas
* Relatórios básicos de vendas

Essas funcionalidades representam processos comuns encontrados em sistemas ERP utilizados por empresas.
